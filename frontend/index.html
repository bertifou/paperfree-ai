Amélioration de la qualité OCR grâce au LLM, et option d'analyse directe par vision multimodale.

🧹 Correction OCR par LLM

Le texte extrait par Tesseract est envoyé au LLM pour corriger les erreurs typiques (l/1, 0/O, mots coupés…)

80%

En dessous de ce seuil, la correction LLM est automatiquement appliquée. Au-dessus, une légère passe de nettoyage est faite si la correction est activée.

💡 Indépendant de la vision : cette correction fonctionne que la vision soit activée ou non. Quand la vision est activée, la voie OCR (b) bénéficiera en plus de la fusion avec le contexte vision.

👁 Analyse par vision (LLM multimodal)

Envoie l'image directement au LLM (bypass OCR). Meilleure précision sur les documents complexes, tampons, manuscrits.

ℹ️ En mode local, le modèle principal doit supporter la vision (ex: llava-v1.6, minicpm-v, moondream2). L'URL et la clé du LLM principal seront utilisées sauf si vous les surchargez ci-dessous.
⚠️ Mode externe : des données (images de vos documents) seront envoyées à un service cloud. Assurez-vous que c'est compatible avec vos obligations de confidentialité.

🔀 Fusion vision × OCR (voie b)

En parallèle de la vision directe, lance Tesseract puis corrige l'OCR en utilisant le contexte JSON de la vision. Les deux JSON sont ensuite fusionnés.

🔬 Pipeline double voie : Voie a) Image → LLM multimodal → JSON  |  Voie b) OCR → correction enrichie → LLM → JSON  →  Merge final

📮 Gmail — OAuth2 obligatoire (gratuit)

Google a supprimé l'auth basique. Il faut créer une app OAuth2 gratuite sur Google Cloud Console (sans carte de crédit, sans vérification pour usage personnel).

  1. Aller sur console.cloud.google.com → Créer un projet (ex: "PaperFree-AI")
  2. APIs & Services → Bibliothèque → chercher "Gmail API" → Activer
  3. APIs & Services → Écran de consentement OAuth → Type : Externe → remplir le nom → Sauvegarder
  4. Credentials → + Créer des identifiants → ID client OAuth 2.0 → Type : Application Web
  5. URI de redirection autorisée : http://localhost:8000/email/oauth/google/callback
  6. Copier le Client ID et le Client Secret ci-dessous → Enregistrer → Connecter mon compte Gmail
  7. Si l'écran "App non vérifiée" apparaît → cliquer Avancé → Accéder à [nom] (non sécurisé) — c'est normal pour une app personnelle.

ℹ️ Serveur IMAP : imap.gmail.com (rempli automatiquement lors de la connexion)

🪟 Outlook / Hotmail / Live — Mot de passe d'application (recommandé)

Microsoft a rendu très difficile l'enregistrement d'une app Azure pour les comptes personnels. La solution la plus simple est un mot de passe d'application, qui fonctionne avec tous les clients email tiers (Thunderbird, Apple Mail, etc.).

✅ Option 1 — Mot de passe d'application (simple, sans Azure)

  1. Aller sur account.microsoft.com/security
  2. Activer la vérification en deux étapes si ce n'est pas déjà fait
  3. Section Sécurité avancée → Mot de passe d'application → Créer
  4. Nommer l'app "PaperFree-AI" → copier le mot de passe généré
  5. Dans la section IMAP ci-dessous : saisir votre adresse email + ce mot de passe
  6. Serveur IMAP : imap-mail.outlook.com

⚙️ Option 2 — OAuth2 via Azure (pour développeurs, carte de crédit requise pour vérification)

  1. Créer un compte Azure sur portal.azure.com (vérification CB, non débité)
  2. Microsoft Entra ID → Inscriptions d'applications → Nouvelle inscription
  3. Type : Comptes Microsoft personnels uniquement · Redirect URI : http://localhost:8000/email/oauth/callback
  4. Certificats & secrets → Nouveau secret → copier la valeur
  5. Autorisations API → Office 365 Exchange Online → IMAP.AccessAsUser.All + offline_access
  6. Remplir les champs OAuth2 Microsoft ci-dessous → Connecter

🏢 Microsoft 365 Entreprise / École — OAuth2 Azure

Votre organisation dispose déjà d'un tenant Azure. Demandez à votre administrateur IT d'enregistrer l'application, ou faites-le vous-même si vous avez les droits.

  1. Aller sur portal.azure.com avec votre compte professionnel
  2. Microsoft Entra ID → Inscriptions d'applications → Nouvelle inscription
  3. Type : Comptes dans cet annuaire uniquement (ou multitenant si plusieurs organisations)
  4. Redirect URI : http://localhost:8000/email/oauth/callback
  5. Certificats & secrets → Nouveau secret client → copier la valeur immédiatement
  6. Autorisations API → Ajouter → Office 365 Exchange Online → Autorisations déléguées → cocher IMAP.AccessAsUser.All + offline_access
  7. Cliquer Accorder le consentement administrateur
  8. Remplir Client ID & Secret dans la section OAuth2 Microsoft ci-dessous → Connecter

ℹ️ Serveur IMAP : outlook.office365.com
